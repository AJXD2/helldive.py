    Models¶

    All models use automatic camelCase ↔ snake_case aliasing via Pydantic, so API responses deserialize transparently.

    helldivepy.models.War ¶

    Bases: APIModel

    Global information of the ongoing war.

    Attributes¶

    now instance-attribute ¶

    now: datetime

    Snapshot timestamp.

    client_version instance-attribute ¶

    client_version: str

    Minimum game version required.

    factions instance-attribute ¶

    factions: list[str]

    Participating factions.

    impact_multiplier instance-attribute ¶

    impact_multiplier: float

    Mission impact calculation factor.

    statistics instance-attribute ¶

    statistics: Statistics

    Aggregated warfare metrics.

    helldivepy.models.Statistics ¶

    Bases: APIModel

    Contains statistics of missions, kills, success rate etc.

    Attributes¶

    mission_time instance-attribute ¶

    mission_time: int

    Total planetside time in seconds.

    bullets_fired instance-attribute ¶

    bullets_fired: int

    Total rounds discharged.

    bullets_hit instance-attribute ¶

    bullets_hit: int

    Total rounds on target.

    time_played instance-attribute ¶

    time_played: int

    Total playtime including off-planet in seconds.

    deaths instance-attribute ¶

    deaths: int

    Human casualties.

    friendlies instance-attribute ¶

    friendlies: int

    Friendly fire casualties.

    accuracy instance-attribute ¶

    accuracy: float

    Average marksmanship percentage.

    player_count instance-attribute ¶

    player_count: int

    Current active participants.

    Comprehensive planetary data integrating all available information.

    index instance-attribute ¶

    ArrowHead planet identifier.

    sector: str

    Geographic sector classification.

    hash instance-attribute ¶

    hash: int

    ArrowHead internal identifier.

    position instance-attribute ¶

    position: Position

    Map coordinates.

    waypoints instance-attribute ¶

    waypoints: list[int]

    Connected planet indices.

    max_health instance-attribute ¶

    max_health: int

    Liberation capacity.

    health instance-attribute ¶

    health: int

    Current liberation status.

    disabled instance-attribute ¶

    disabled: bool

    Whether the planet is currently inactive.

    regen_per_second instance-attribute ¶

    regen_per_second: float

    Uncontested enemy regeneration rate.

    Planetary warfare metrics.

    regions instance-attribute ¶

    regions: list[Region]

    Subdivisions with individual health and availability status.

    helldivepy.models.Assignment ¶

    Bases: APIModel

    Directive issued to Helldivers by high command (a.k.a. Major Order).

    Attributes¶

    progress instance-attribute ¶

    progress: list[int]

    Per-task advancement values; injected into each Task during validation.

    reward class-attribute instance-attribute ¶

    reward: Reward | None = None

    Primary reward (deprecated in favour of rewards).

    helldivepy.models.Task ¶

    Bases: APIModel

    Constituent objective within an assignment requiring completion.

    Note: type and the keys of values are community-reverse-engineered (TaskType, TaskValueType) and may be incomplete. Unknown values fall back to int.

    Attributes¶

    type instance-attribute ¶

    type: TaskType | int

    Objective classification (community-reverse-engineered, may be incomplete).

    values instance-attribute ¶

    values: dict[TaskValueType | int, int]

    Parsed valueType → value mapping. Keys are TaskValueType where known, else raw int.

    progress class-attribute instance-attribute ¶

    progress: int = 0

    Raw progress value injected by Assignment during validation.

    goal property ¶

    goal: int | None

    The target value for this task (kill count, liberation %, etc.)

    progress_perc property ¶

    progress_perc: float | None

    Progress towards goal as a percentage (0.0–100.0), or None if not applicable.

    is_liberation_task property ¶

    is_liberation_task: bool

    Liberation/defense tasks track progress via Campaign health, not Assignment.progress.
